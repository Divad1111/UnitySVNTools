# 需求

核心目标是在Unity中直接操作SVN的相关功能，不想跳转到Unity外部去操作SVN。你可以参考visual studio code的Git窗口布局，其他Git特有的功能，保留SVN的相关功能。

在Unity中新增一个EditorWindow窗口，窗口分四个区域从上到下分别为：
1. 标题栏：左边显示当前SNV仓库路径，右边显示三个点，点击后显示上下文菜单，菜单里有一个更新按钮，点击后可以更新当前SVN仓库的文件。
2. 提交日志输入区域：一个标题栏（标题栏右边有一个按钮，配置点击后打开一个配置窗口，里面可以配置忽略目录，忽略目录下的修改不显示在改变列表中），一个日志输入框，输入框下面是一个提交按钮，点击提交后可以提交当前改变的文件（如果有选中的则只提交选中的，没有则提交所有修改的文件，所有修改的文件并不是当前SVN仓库下的所有修改文件，因为会加过滤目录，过滤目录中的文件不提交）。
3. 当前改变文件列表：显示除忽略目录下的所有工作区修改的文件，包括新增、修改、删除，并在每个文件后面显示对应状态（A，U，D），且在选中每个文件项后在右边显示一个还原按钮，双击这个文件可以通过TortoiseSVN的配置打开diff和在TortoiseSVN中显示日志是打开的一样。
4. 状态栏：显示SVN的相关状态信息，比如：当前分支，当前修改的文件数量（排除忽略目录下的文件）等。


配置窗口：
1. 窗口标题：忽略目录配置
2. 窗口内容：忽略目录列表，右边有个添加目录，点击后可以选择忽略目录。添加后则在忽略列表中显示。
